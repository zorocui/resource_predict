# ECharts 默认七天窗口设计

## 目标

资源详情中的 ECharts 图表默认激活 `7d` 时间范围。用户仍可切换 `24h`、`3d` 和“全部”，该调整仅影响图表响应与展示，不影响预测、决策、误差报告和调配门控。

## 数据加载

- 前端初始状态将 `chartRangeKey` 设为 `7d`。
- 七天窗口沿用现有扩展窗口请求，`history_points` 上限为 10000。
- 响应仍在服务端完成时间范围过滤，测试窗口和未来预测保持完整。
- 不先请求 1000 点再补请求，避免首次打开产生两次图表请求。
- 不提供无边界超大响应；“全部”同样受 10000 点上限保护。

## 交互

- 首次进入指标页时，`7d` 按钮处于激活状态。
- 切换指标或容器时继续使用当前时间范围，并沿用按指标、按容器的请求缓存键。
- 用户切换到其他范围后，本次详情会话保持其选择。

## 验证

- JavaScript 语法检查通过。
- 浏览器刷新后首次进入指标页，确认 `7d` 激活且图表成功绘制。
- 确认请求采用有界历史点数，浏览器控制台无错误。
- 运行完整 Python 回归，确保后端详情、预测和调配行为不变。
