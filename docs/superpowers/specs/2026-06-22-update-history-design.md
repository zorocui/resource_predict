# 数据更新历史记录设计

## 背景

当前数据更新状态只保存在进程内的 `_update_status` 字典中。`GET /api/update-status` 和“数据更新”页面只能看到当前或最近一次任务；应用重启后这些信息会丢失，也无法回看更早的更新结果。

## 目标

- 在现有“数据更新”页面查看此前的更新记录。
- 成功和失败的更新均记录。
- 记录跨应用重启保留。
- 最多保留最近 100 条，避免文件无限增长。
- 保持当前状态卡片和轮询行为不变。

## 非目标

- 不提供统计图表、筛选、搜索或复杂分页。
- 不记录任务执行过程中的每个阶段变更。
- 不引入数据库或新的外部服务。

## 存储设计

历史记录保存在 `outputs/update_history.json`。文件是一个 JSON 对象，包含版本号和按完成时间倒序排列的记录数组：

```json
{
  "version": 1,
  "records": [
    {
      "id": "update-1782115200000-1",
      "status": "success",
      "phase": "idle",
      "task_source": "页面手动拉取",
      "fetch_window_label": "增量窗口：最近 7 小时",
      "started_at": 1782115100.0,
      "finished_at": 1782115200.0,
      "elapsed_seconds": 100.0,
      "resources_updated": 12,
      "resources_created": 1,
      "predicted_resources": 13,
      "total_new_points": 3744,
      "message": "K8S Prometheus 数据拉取完成",
      "error": null
    }
  ]
}
```

每次任务进入成功或失败终态时，基于当前状态和任务结果生成一条不可变记录，插入数组头部并截取前 100 条。写入使用同目录临时文件加 `os.replace()`，避免进程中断留下半文件。历史读写使用独立线程锁，且历史写入失败只记录错误日志，不改变更新任务本身的结果。

文件不存在时返回空记录。文件内容损坏或结构不合法时记录错误日志并返回空记录，不阻塞状态接口和更新流程。

## 更新链路

所有更新入口均复用统一的历史记录函数：

- VM 手动 Pull、后台定时 Pull。
- `POST /api/update-data` 推送更新。
- `POST /api/upsert-data` 推送新增或更新。
- K8S Prometheus 页面手动拉取和后台定时拉取。

开始任务时继续更新 `_update_status`，并补齐稳定的任务来源。任务成功或失败时先形成终态快照，再写入历史。K8S 外部拉取使用现有 `mark_external_update_finished()` / `mark_external_update_failed()` 接入；普通更新在 `_do_update()` 的成功和异常分支接入。一个任务只写一条终态记录，K8S 包装层不得对 `_do_update()` 已记录的同一任务重复写入。

## API

新增：

```text
GET /api/update-history?limit=20
```

- `limit` 默认 20，允许 1–100。
- 非法或越界参数返回 HTTP 400。
- 响应结构为 `{"records": [...], "count": n, "retention_limit": 100}`。
- 记录按完成时间从新到旧排列。

现有 `GET /api/update-status` 响应保持兼容。

## 页面

“数据更新”页面保留当前状态卡片和消息区，在其下增加“历史更新记录”区域。默认加载最近 20 条，每条显示：

- 成功或失败状态。
- 任务来源和拉取窗口。
- 开始与结束时间、耗时。
- 更新资源、新增资源、新增数据点和预测资源数量。
- 失败原因或完成消息。

没有历史时显示简洁空状态。进入“数据更新”页面时同时请求当前状态和历史；轮询发现任务结束后刷新历史。历史请求失败只在历史区域显示错误，不影响当前状态展示。

## 测试

- 成功任务写入完整历史记录。
- 失败任务写入错误记录。
- 历史记录跨重新实例化读取，按新到旧排序。
- 超过 100 条时只保留最新 100 条。
- 损坏文件不阻塞读取或更新主流程。
- API 默认条数、指定条数及非法 `limit`。
- 页面历史空状态、成功/失败记录渲染和任务结束后的刷新行为。
